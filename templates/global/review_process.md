## General Review Process

When reviewing code in this repository, always evaluate:

1. **Correctness** — logic, edge cases, error handling  
2. **Security** — input validation, data handling, secrets, injection risks  
3. **Performance** — unnecessary allocations, loops, I/O, concurrency  
4. **Testability** — unit tests, mocks, coverage, deterministic behavior  
5. **Coding Standards** — naming, structure, formatting, idioms  
6. **Maintainability** — readability, modularity, documentation
